# Portfolio 最终 PRD

## 1. 项目定位

Portfolio 是一个面向实习投递的静态个人作品集网站，核心目标是让招聘方快速理解我的 AI 应用开发方向、CodePilot 项目能力和基础前端工程能力。

注意：

- Portfolio 是辅助展示载体。
- CodePilot 是当前核心项目。
- Portfolio 当前不是 AI Agent 项目。
- AI Agent 能力可以作为后续方向，但不进入当前 MVP。

## 2. 项目背景

当前用户正在准备实习投递，需要一个统一入口展示：

- 求职方向
- 核心项目 CodePilot
- 基础前端工程能力
- 可上线、可维护、可迭代的项目交付能力

简历和 GitHub 信息相对分散，Portfolio 用于补充展示，不替代简历和 CodePilot 本身。

## 3. 目标用户

### 3.1 HR

关心：

- 这个人是谁
- 投什么方向
- 是否有可查看的项目
- 是否方便联系

### 3.2 技术面试官

关心：

- CodePilot 是什么
- 技术栈是否真实
- 项目是否能讲清楚
- 是否具备基础工程能力

### 3.3 实习招聘方

关心：

- 是否适合 AI 应用开发 / Agent 开发 / Python 后端方向
- 是否能快速看到项目亮点
- 是否能跳转到 GitHub 查看代码

## 4. 核心目标

按优先级排序：

1. 5 秒内说明我是谁、投什么方向。
2. 重点讲清楚 CodePilot。
3. 提供 Email 和 GitHub 入口。
4. 展示基础前端工程实现能力。
5. 完成后可上线，用于近期实习投递。

## 5. 第一版 MVP 范围

第一版只做 4 个模块：

```txt
Hero
About
CodePilot
Contact
```

不新增其他模块。

---

## 6. 模块需求

### 6.1 Hero

目标：

让用户在第一屏快速知道：

- 我是谁
- 我的求职方向
- 我的核心项目是什么

内容方向：

```txt
2027 届人工智能本科生，聚焦 AI 应用开发 / Agent 开发 / Python 后端。
当前核心项目是 CodePilot，一个面向 GitHub 仓库理解与代码审查的 AI 应用项目。
```

要求：

- 中文为主，技术关键词保留英文。
- 不夸大，不写"精通"。
- 不写虚假数据。
- 不放没有准备好的简历下载按钮。

### 6.2 About

目标：

简短说明个人背景和技术关注点。

必须表达：

- 2027 届人工智能本科生
- 求职方向：AI 应用开发 / Agent 开发 / Python 后端
- 当前重点项目：CodePilot
- 关注真实工程化、小步迭代、可上线项目

不要写：

- 多年经验
- 精通全栈
- 夸张自我评价
- 与当前 MVP 无关的经历

### 6.3 CodePilot

目标：

作为页面核心内容，讲清楚 CodePilot。

必须展示：

- 项目名称：CodePilot
- 项目定位：GitHub 仓库理解与 AI 代码审查应用
- 解决的问题：帮助用户理解仓库结构并生成代码审查报告
- 基本流程：输入 GitHub 仓库链接 → 解析源码结构 → 构建结构化上下文 → 生成审查报告
- 技术栈：Python / FastAPI / React / Tree-sitter / LLM API / Markdown 报告生成
- 当前状态：以真实情况为准，不夸大
- GitHub 链接：https://github.com/Strange-Men/CodePilot
- Demo：TBD，不编造

展示深度：

第一版做简洁但清楚的项目展示，不展开过长版本迭代，不写虚假指标。

### 6.4 Contact

目标：

提供真实联系入口。

第一版只放：

- Email：1643229721@qq.com
- GitHub：Strange-Men

要求：

- 不放未确认平台。
- 不放简历下载按钮。
- 不写 Resume: TBD。
- 后续准备好真实简历 PDF 后再加。

## 7. 用户使用流程

1. 用户打开 Portfolio 首页。
2. 在 Hero 判断我的方向是否匹配。
3. 查看 About 了解背景。
4. 查看 CodePilot 判断项目能力。
5. 点击 GitHub 查看代码，或通过 Email 联系我。

## 8. 输入与输出

### 8.1 输入

第一版不需要用户输入。

不做：

- 表单
- 登录
- AI 问答输入
- 数据提交

### 8.2 输出

页面向用户输出：

- 个人定位
- 求职方向
- CodePilot 项目信息
- GitHub 链接
- Email 联系方式

## 9. 语言策略

第一版使用：

```txt
中文为主，技术名词保留英文
```

原因：

- 当前主要服务国内实习投递。
- 中文表达更直接。
- 技术关键词保留英文，方便面试官识别。

## 10. 第一版明确不做

第一版不做：

- AI Agent
- AI 问答
- 后端
- 登录
- 数据库
- 博客系统
- 多项目堆叠
- Skills 大而全展示
- Workflow 详细展示
- Portfolio 开发过程展示
- ReactBits
- WebGL
- cursor spotlight
- 复杂动效
- 简历下载按钮
- 未确认 Demo 链接
- 虚假数据
- 虚假经历
- 虚假项目

## 11. 后续版本可以考虑

后续可以考虑，但不进入当前 MVP：

- AI Agent 能力
- 简历 PDF 下载
- Skills 简洁展示
- Workflow 小步迭代展示
- Portfolio 开发过程展示
- 轻量动效
- 响应式增强
- Vercel 部署
- CodePilot Demo 链接

任何后续扩展都必须在第一版 MVP 完成后再评估。

## 12. 非功能要求

### 12.1 真实性

- 所有项目、链接、数据必须真实。
- 未确认信息写 TBD 或不展示。
- 不允许编造经历、指标、Demo。

### 12.2 可用性

- 页面结构必须清楚。
- 用户 5 秒内能理解方向。
- CodePilot 信息必须容易阅读。
- Contact 入口必须可点击。

### 12.3 性能

- 第一版优先静态页面。
- 不引入复杂动效。
- 不使用 WebGL。
- 不使用视频背景。
- 不使用重型动画库。

### 12.4 可维护性

- 内容和组件后续应可拆分。
- 项目文档必须持续更新。
- 每轮开发必须可验收、可回滚。

## 13. 验收标准

最终 MVP 完成时必须满足：

1. 页面只包含 Hero、About、CodePilot、Contact 四个核心模块。
2. 第一屏能看出我是 2027 届人工智能本科生，方向是 AI 应用开发 / Agent 开发 / Python 后端。
3. CodePilot 是页面核心内容，不被视觉或动效抢走重点。
4. CodePilot GitHub 链接真实可点击。
5. Email 真实展示。
6. 没有简历 PDF 时，不出现简历下载按钮。
7. 没有编造 Demo、数据、经历。
8. 没有出现禁用项目名。
9. 不包含 AI 问答、后端、登录、数据库。
10. 项目可以进入后续设计基调阶段。

## 14. 风险与规避

### 风险 1：CodePilot 讲不清

规避：

CodePilot 模块优先写清楚定位、流程、技术栈和链接。

### 风险 2：页面好看但内容空

规避：

视觉只能服务内容，不能反客为主。

### 风险 3：过度包装

规避：

Portfolio 只定位为静态作品集，不包装成 AI Agent 项目。

### 风险 4：范围再次扩大

规避：

当前 MVP 锁定 4 个模块，不新增 Skills、Workflow、Blog。

### 风险 5：链接不存在

规避：

CodePilot GitHub 使用真实链接。Demo 不存在就写 TBD 或不展示。

### 风险 6：简历入口不完整

规避：

没有真实 PDF 前不放下载按钮。

### 风险 7：为了视觉拖慢投递

规避：

优先能上线、能讲清，再做视觉细节。

## 15. 最终结论

Portfolio 第一版可以进入下一阶段。

下一阶段不是开发，而是：

```txt
Phase 3-1：确定设计基调
```

最终 PRD 已固化以下核心边界：

- 当前只做静态作品集网页
- 当前只展示 Hero / About / CodePilot / Contact
- 当前优先上线和实习投递
- 当前不做 AI Agent / 后端 / AI 问答
- CodePilot 是核心项目
- Portfolio 是展示载体
