# Workflow Lock

本项目必须严格按照"九准备五关键"分轮执行，不能一个 Prompt 跑完。

## 开发前九准备

1. 明确原始想法
2. 想法压力测试
3. 输出想法判断结论
4. 生成初版 PRD
5. 追问对齐 PRD
6. 固化最终 PRD
7. 确定设计基调
8. 规划页面结构
9. 规划核心交互流程

以上 9 步必须分多个 Prompt 完成。

未完成 PRD 对齐，不允许进入代码开发。
未完成设计基调，不允许规划页面。
未完成页面结构，不允许初始化工程。
未完成项目边界、非功能需求、技术栈、轻量架构和 Git 回滚策略，不允许写代码。

## 打地基

写代码前还必须完成：

1. 明确项目边界
2. 明确非功能需求
3. 锁定技术栈
4. 设计轻量架构
5. 建立全局上下文文档
6. 固化开发规范
7. 指定参考文件夹
8. 建立 Git 回滚策略

## 开发中五关键

1. 小步迭代
2. 主动模块拆分
3. 状态摘要
4. 安全底线
5. 科学检查报错

## 硬规则

- 每轮只做一个小任务。
- 每轮必须明确目标、边界、修改范围、禁止范围、验收标准。
- 每轮必须可验收、可回滚、可复盘。
- 不允许一次性生成完整项目。
- 不允许 AI 自由发挥。
- 不允许编造项目、链接、数据和经历。
- 当前作品集只允许讨论 CodePilot 和 Portfolio。
