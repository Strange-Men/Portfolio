# Portfolio 非功能需求

## 1. 非功能需求目标

Portfolio 第一版是面向实习投递的静态作品集网站。

非功能目标：

- 真实可信
- 加载快
- 易阅读
- 易联系
- 移动端可用
- 可上线
- 可维护
- 不引入不必要成本
- 不为炫技牺牲内容

当前优先级：

```txt
真实性 > 内容清晰 > 可用性 > 性能 > 视觉细节 > 动效
```

---

## 2. 安全要求

### 2.1 密钥与敏感信息

当前第一版是静态页面。

要求：

- 不出现 API Key
- 不提交 `.env`
- 不提交私密配置
- 不在前端写任何密钥
- 不接入需要密钥的后端服务
- 不接入未确认的第三方统计脚本

### 2.2 链接安全

所有外链必须真实。

当前允许链接：

- GitHub 主页：`https://github.com/Strange-Men`
- CodePilot GitHub：`https://github.com/Strange-Men/CodePilot`
- Email：`mailto:1643229721@qq.com`

要求：

- 不放空链接
- 不放假链接
- Demo 未确认时不展示 Demo 按钮
- 简历 PDF 未确认时不展示下载按钮
- 外链后续实现时应使用安全打开方式，例如 `rel="noreferrer noopener"`

### 2.3 内容安全

不允许：

- 编造经历
- 编造项目
- 编造 Demo
- 编造数据
- 编造效率提升指标
- 写无法证明的奖项或成果
- 把 Portfolio 包装成当前 AI Agent 项目

---

## 3. 性能要求

### 3.1 页面类型

第一版必须优先保持静态页面。

要求：

- 不需要后端请求
- 不需要数据库
- 不需要登录
- 不需要 AI 问答请求
- 不需要实时数据加载

### 3.2 首屏性能

Hero 是第一屏，必须优先可读。

要求：

- 核心文字不能依赖图片加载后才显示
- 不使用视频背景
- 不使用 WebGL
- 不使用重型动画库
- 不使用复杂滚动监听
- 不做 cursor spotlight
- 不做 canvas mask

### 3.3 资源控制

后续实现时需要控制：

- 图片体积
- 字体加载数量
- CSS 复杂度
- JS 体积
- 动画数量

如果使用 Google Fonts，只允许少量字体，且不能影响核心文字展示。

### 3.4 构建要求

后续进入代码阶段后，每轮必须能运行：

```bash
npm run build
```

构建失败不能进入下一轮。

---

## 4. 可用性要求

### 4.1 信息可读

页面必须让用户快速理解：

1. 我是谁
2. 投什么方向
3. CodePilot 是什么
4. 如何联系我

要求：

- Hero 5 秒内可理解
- About 不写长篇
- CodePilot 是核心展示
- Contact 清晰可点击

### 4.2 CTA 清晰

核心 CTA：

- 查看 CodePilot
- GitHub
- 联系我
- CodePilot GitHub

要求：

- 每个 CTA 都有明确目标
- 不出现空按钮
- 不出现 TBD 按钮
- 不出现未确认 Demo 按钮
- 不出现简历下载按钮

### 4.3 移动端可用

移动端必须满足：

- 不出现横向滚动
- 文字不挤压
- CTA 可点击区域足够大
- CodePilot 信息单列展示
- Contact 容易点击和复制

---

## 5. 可访问性要求

后续实现时必须考虑：

- 语义化 HTML
- 按钮和链接可聚焦
- focus-visible 可见
- 文本对比度足够
- 链接文案清楚
- Email 和 GitHub 不只靠图标表达
- 动效必须尊重 `prefers-reduced-motion`

不允许：

- 只有图标没有文字说明
- hover 才能看到关键信息
- 动效影响阅读
- 颜色对比过低

---

真实性是第一优先级。

必须真实展示：

- Email：`1643229721@qq.com`
- GitHub：`Strange-Men`
- CodePilot GitHub：`https://github.com/Strange-Men/CodePilot`

未确认内容处理：

- 简历 PDF：不展示下载按钮
- 其他平台：不展示
- 未确认数据：不展示
- 未确认经历：不展示

---

## 7. 成本要求

当前 MVP 成本必须接近 0。

要求：

- 不使用付费服务
- 不调用 LLM API
- 不部署后端
- 不使用数据库
- 不接入收费第三方服务
- 静态部署优先

后续部署可以优先考虑：

- Vercel
- GitHub Pages

但部署平台在后续阶段单独确认。

---

## 8. 维护性要求

后续实现时应保证：

- 内容和组件分离
- Hero / About / CodePilot / Contact 可以独立维护
- 链接集中管理
- 项目信息集中管理
- 样式不要全部堆在一个超大文件里
- 每轮修改可追踪、可回滚

不允许：

- 一坨代码
- 一次性大范围重构
- 修改无关文件
- 未提交状态下连续开发多轮

---

## 9. 兼容性要求

第一版至少考虑：

- 桌面端
- 笔记本
- 手机端

后续实现时重点检查：

```txt
375px
430px
768px
1024px
1440px
1920px
```

要求：

- 不横向溢出
- Hero 不挤压
- CodePilot 可阅读
- Contact 可点击

---

## 10. 动效约束

MVP 阶段不做复杂动效。

允许后续考虑：

- hover transition
- focus-visible
- 轻微 fade-in
- 简单按钮反馈

当前禁止：

- ReactBits
- GSAP
- Framer Motion
- WebGL
- cursor spotlight
- canvas mask
- 视频背景
- 复杂滚动动画
- 大量视差效果

动效原则：

```txt
可读性优先，内容优先，CodePilot 优先。
```

---

## 11. 部署前要求

第一版上线前必须满足：

1. `npm run build` 通过
2. 页面只包含 Hero / About / CodePilot / Contact
3. 所有真实链接可点击
4. 没有空链接
5. 没有假链接
6. 没有未确认 Demo
7. 没有简历下载按钮
8. 没有禁用项目名
9. 移动端不崩
10. Git 状态干净

---

本阶段完成后，必须满足：

1. 安全要求明确。

3. 可用性要求明确。
4. 可访问性要求明确。
5. 真实性要求明确。
6. 成本边界明确。
7. 动效边界明确。
8. 部署前检查项明确。
9. 不进入代码开发。
10. 可以进入下一阶段：锁定技术栈。
