# Portfolio 初版 PRD

## 1. 项目背景

用户正在准备实习投递，目标是进入 AI 应用开发 / Agent 开发 / Python 后端方向的岗位。

当前问题：

- 简历和 GitHub 信息分散，招聘方需要自己去翻找。
- 缺少一个统一入口，让招聘方快速了解求职方向和核心项目。
- CodePilot 是核心项目，但 GitHub 仓库本身不能直观展示项目价值。

**Portfolio 是辅助展示载体，不是核心 AI 项目。** 它的价值在于把已有的项目和背景信息包装成可浏览的内容，降低招聘方的了解成本。

## 2. 项目定位

Portfolio 是一个面向实习投递的静态个人作品集网站，核心目标是让招聘方快速理解我的 AI 应用开发方向、CodePilot 项目能力和基础前端工程能力。

## 3. 目标用户

### HR

最关心：

- 候选人的项目经历是否真实
- 是否有可访问的在线链接
- 页面是否能快速加载、信息是否清晰
- "这个人做过什么"

### 技术面试官

最关心：

- 项目的技术深度和实现细节
- 是否有可演示的 Demo
- 代码质量（会去 GitHub 看）
- 技术选型的理由
- 遇到了什么问题、怎么解决的

### 实习招聘方

最关心：

- 是否有完成度高的项目
- 是否有持续学习和迭代的证据
- 是否有工程化意识（不只是写代码）

### 查看作品链接的人

最关心：

- 快速了解这个人是谁、做什么方向
- 有没有值得看的项目
- 怎么联系

## 4. 核心目标

按优先级排序：

1. **5 秒内说明我是谁、投什么方向。** 首屏必须让用户快速判断匹配度。
2. **重点展示 CodePilot。** 这是作品集存在的根本理由。
3. **提供 GitHub / Email / 简历入口。** 投递必需的联系方式。
4. **展示基础前端工程实现能力。** 技术栈实践证明。
5. **为后续上线和迭代留空间。** 第一版先上线，后续再扩展。

## 5. MVP 范围

第一版只做 4 个模块：

### 5.1 Hero

**目标：** 让用户快速知道我是谁、投什么方向、最核心的项目是什么。

**内容包括：**

- 姓名 / 身份（2027 届人工智能本科生）
- 求职方向（AI 应用开发 / Agent 开发 / Python 后端）
- 一句话介绍（聚焦 AI 应用开发，当前核心项目是 CodePilot）
- CTA 按钮：查看 CodePilot / 联系我 / GitHub

**说明：** Hero 只需要一句话 + 视觉基调，不要堆砌内容。

### 5.2 About

**目标：** 简短说明背景、求职方向和技术关注点。

**内容包括：**

- 2027 届人工智能本科生
- 方向聚焦 AI 应用开发 / Agent 开发 / Python 后端
- 当前核心项目是 CodePilot
- 关注工程化、小步迭代和真实可上线项目

**说明：** About 要精炼，3-5 句话足够，不要写成长篇自传。

### 5.3 CodePilot

**目标：** 作为页面核心内容，讲清楚 CodePilot 是什么、解决什么问题、怎么实现的。

**必须包含：**

- **项目定位：** CodePilot 是什么，面向什么用户
- **解决的问题：** 用户在什么场景下遇到什么痛点
- **用户输入：** 用户给 CodePilot 什么
- **系统输出：** CodePilot 返回什么
- **核心流程：** 从输入到输出的主要步骤
- **技术栈概述：** 用了什么技术，为什么选这些技术
- **当前进展：** 项目处于什么阶段，已完成什么、待完成什么
- **可讲的技术点：** 面试时可以展开讲的技术难点或亮点
- **GitHub / Demo 链接占位：** 如果真实链接不存在，用 TBD，不要编造

**说明：** CodePilot 是作品集的核心，必须讲清楚。如果 CodePilot 本身讲不清楚，作品集就失去了存在的理由。

### 5.4 Contact

**目标：** 提供下一步联系入口。

**内容包括：**

- Email
- GitHub
- 简历入口（如果简历 PDF 不存在，先写 TBD，不做下载按钮）

**说明：** Contact 保持最简，不要堆砌社交链接。

## 6. 用户使用流程

1. 用户打开首页。
2. 先看 Hero，判断方向是否匹配。
3. 滚动到 About，了解背景。
4. 查看 CodePilot，判断项目能力。
5. 点击 GitHub / Email / 简历入口。

**说明：** 流程要简单直接，不要设置复杂的跳转或多层页面。

## 7. 输入与输出

### 输入

用户无需输入表单。Portfolio 是纯展示页面，不需要用户交互。

### 输出

页面向用户输出：

- 个人定位信息（我是谁、做什么方向）
- CodePilot 项目信息（是什么、怎么实现、有什么亮点）
- 联系方式（Email、GitHub）
- 外部跳转入口（GitHub 仓库、Demo 链接、简历 PDF）

## 8. 验收标准

第一版必须满足以下标准：

- [ ] 页面第一屏能看出 AI 应用开发方向。
- [ ] 页面只展示 Hero、About、CodePilot、Contact 四个核心模块。
- [ ] CodePilot 介绍足够清楚，不只是项目名。
- [ ] 没有编造链接、数据、经历。
- [ ] 没有出现禁用项目名（ProfileAgent、CustomerOps、数字孪生、智慧校园）。
- [ ] 不包含 AI 问答、后端、登录、数据库功能。
- [ ] 第一版可以作为实习投递作品集入口。
- [ ] 后续可以继续做设计基调和页面结构规划。

## 9. 暂不实现范围

第一版明确不做：

- AI 问答
- 后端
- 登录
- 数据库
- 博客系统
- 多项目堆叠
- Skills 大而全展示
- Workflow 详细展示
- ReactBits
- WebGL
- cursor spotlight
- 复杂动效
- Portfolio 开发过程展示

**说明：** 这些内容不是永久不做，而是第一版先不做。先完成核心内容，再考虑扩展。

## 10. 风险点

### 风险 1：CodePilot 讲不清

- **表现：** 面试时被问到 CodePilot 的实现细节，答不上来。
- **规避：** 在做作品集之前，先写一份 CodePilot 的技术说明文档，梳理清楚架构和亮点。

### 风险 2：内容空洞

- **表现：** 视觉设计不错，但 CodePilot 讲不清楚，项目经历描述空洞。
- **规避：** 先整理 CodePilot 的技术架构、核心功能、遇到的难点，再开始做页面。

### 风险 3：过度包装

- **表现：** 把静态页面包装成"AI 驱动"，把用 Claude Code 写代码包装成"革命性工作流"。
- **规避：** 如实描述，用词准确，不要夸大。

### 风险 4：视觉大于内容

- **表现：** 花大量时间做动画效果，但内容没准备好。
- **规避：** 第一版不做复杂动效，先完成内容，再考虑视觉优化。

### 风险 5：链接不存在

- **表现：** 作品集里放了项目链接，但打不开或者不是真实项目。
- **规避：** 确保 CodePilot 有可访问的 Demo 或 GitHub 仓库。如果不存在，用 TBD 占位。

### 风险 6：项目范围再次扩大

- **表现：** 开发过程中增加新模块，导致 MVP 失控。
- **规避：** 严格控制 MVP 范围，不要在开发过程中增加新模块。

### 风险 7：页面做完但不上线

- **表现：** 做了很久但没有部署，投递时没有可访问的链接。
- **规避：** 第一版用最简单的部署方式（GitHub Pages 或 Vercel），先上线再说。

### 风险 8：花太多时间在动效

- **表现：** 反复调整动画效果，迟迟不定稿。
- **规避：** 第一版不做复杂动效，先完成内容，再考虑视觉优化。

## 11. MVP 边界

**当前 MVP 是静态作品集页面，不是 AI 应用、不是后端项目、不是博客系统、不是全功能个人网站。**

具体边界：

- **是：** 静态页面、React + Vite + Tailwind、展示 CodePilot、提供联系方式
- **不是：** AI 问答、后端 API、登录系统、数据库、博客、多项目展示、复杂动效

## 12. 后续待对齐问题

下一步 PRD 追问时需要确认的问题：

1. **Hero 主文案最终怎么写？** 需要确定一句话介绍的具体内容。
2. **CodePilot 展示到什么深度？** 需要确定技术细节的详细程度。
3. **是否现在就准备简历 PDF？** 如果没有简历，Contact 模块的简历入口怎么处理。
4. **GitHub / Demo 链接是否已有？** 如果没有，需要先准备。
5. **Contact 是否只放 Email + GitHub？** 是否需要放其他联系方式。
6. **是否需要中英文双语？** 当前默认中文，是否需要英文版本。
7. **第一版视觉风格是极简还是暗色高级？** 需要确定设计基调。
8. **CodePilot 的技术说明文档是否已准备好？** 这是做作品集的前提。
9. **部署方式选择 GitHub Pages 还是 Vercel？** 需要提前规划。
10. **第一版完成后是否立即投入实习投递？** 需要明确时间节奏。
