## 4. Why OG Image Matters

Portfolio 已经完成页面主体视觉打磨（v1.33.0-visual-identity-polish），但链接发送给 HR / 面试官时，第一眼可能来自社交平台或聊天工具的链接预览。

当前状态：

- 微信 / 飞书 / 钉钉分享链接时，没有图片预览，只有文字标题和描述
- Twitter / LinkedIn 分享时，使用 `summary` 卡片（小图），但没有图片
- 如果社交平台无法抓取到 `og:image`，会降级为纯文本链接

缺少分享图的影响：

- HR 在聊天工具中收到链接时，视觉吸引力低
- 链接预览与页面本身的 Dark Technical Editorial 风格不一致
- 降低了 Portfolio 作为"作品"的整体专业感

---

## 5. Visual Direction

推荐方向：Dark Technical Editorial Preview

关键词：

- Dark — 与页面 `#09090b` 背景一致
- Technical — 体现 AI 工程实践，不是泛泛的设计作品集
- Editorial — 像高级数字杂志的排版，克制、清晰
- Personal Portfolio — 明确是个人作品集，不是公司产品页
- Proof Trail — 延续页面中已有的"验证方式"视觉语言
- Controlled Glow — 紫粉光晕克制使用，不堆叠
- Clear Typography — 中英文清晰可读，1200×630 下不模糊

视觉元素建议：

- 深黑背景 `#09090b`
- 细线网格（延续页面 Hero 的 grid texture）
- 低透明紫粉光晕（延续 `site-ambient-bg` 的 radial-gradient）
- 项目编号 01 / 02 / 03（延续 Projects 区域的大号编号锚点）
- 少量 Proof Trail 线条（延续 proof-block 的左边框视觉语言）
- 无人物、无机器人、无代码雨、无赛博朋克、无霓虹

## 6. Candidate Schemes

### 6.1 Scheme A: Controlled SVG OG Image

说明：

用手写 SVG 制作分享图，尺寸 1200×630。纯代码绘制暗色背景、细线、项目编号和文字。

优点：

- 清晰：SVG 是矢量格式，任意缩放不模糊
- 小体积：预计 5-15 KB，远小于 PNG / WebP
- 风格可控：每一像素都在代码中，不会出现 AI 生成的脏边或伪影
- 适合当前克制高级风格：Dark Technical Editorial 的最佳载体
- 可版本控制：SVG 是纯文本，diff 可读
- 可复用：同一张图可同时用于 README banner
- 无 AI 味：不会让人觉得是"AI 生成的海报"

缺点：

- 氛围感弱于复杂图片：无法实现照片级纹理或 3D 效果
- 部分平台对 SVG OG 图支持有限：需要验证微信、飞书等国内平台是否正确渲染

风险：

- 低。SVG 是成熟方案，主要风险是国内聊天平台的兼容性（可通过导出 PNG 兜底）。

是否推荐：**是，优先推荐。**

---

### 6.2 Scheme B: Image2 Texture + Text Overlay

说明：

先用 Image2 生成暗色 proof-trail 背景纹理（WebP 格式），再用代码或设计工具叠加文字。

适合生成：

- 暗色技术纹理
- 细线节点
- 低透明 proof trail
- 无文字背景

不适合生成：

- 完整带文字海报（AI 生成文字容易变形）
- 人物图
- 机器人图
- 3D 场景
- 代码瀑布
- 霓虹城市
- 复杂科技海报

优点：

- 氛围更强：AI 生成的暗色纹理可以提供更丰富的视觉层次
- 可与 SVG 文字叠加：背景用 WebP，文字用 SVG / HTML overlay

缺点：

- 有 AI 图风险：如果纹理过于明显，可能被识别为 AI 生成
- 体积风险：WebP 背景图可能 50-200 KB，增加页面加载
- 风格不可控：Image2 生成结果有随机性，可能与页面风格不一致
- 需要额外工具链：生成 → 筛选 → 裁剪 → 叠加文字 → 导出

风险：

- 中。AI 生成纹理可能与页面克制风格冲突，需要多次尝试筛选。

是否推荐：**否，不作为首选。** 如果后续需要更强氛围，可以作为 Scheme A 的补充——用 Image2 生成背景纹理，再叠加 SVG 文字。

---

### 6.3 Scheme C: No OG Image Yet

说明：

保持当前无 `og:image` 状态，后续再做。

判断：

**不推荐。** 当前 Portfolio 已经进入视觉打磨阶段（v1.33.0），缺少 `og:image` 会削弱链接分享第一印象。v1.30.0 时因为没有合适图片资源而跳过，现在页面视觉已经成熟，有条件制作匹配的分享图。

是否推荐：**否。**

---

## 7. Recommended Scheme

推荐优先使用 Scheme A：Controlled SVG OG Image。

建议先制作 `public/og/portfolio-og.svg`，用纯 SVG 绘制暗色背景、细线、项目编号和文字。

理由：

1. 当前页面风格是 Dark Technical Editorial，适合克制、清晰、可控的 SVG
2. 不希望引入 AI 大图导致风格变脏
3. SVG 体积小（5-15 KB），适合提交到 `public/``
4. 后续可以用同一张图作为 README banner
5. 如果未来需要更强氛围，再单独生成 WebP 背景替换

如果平台兼容性需要，从 SVG 导出 PNG 兜底：

- `public/og/portfolio-og.svg` — 优先
- `public/og/portfolio-og.png` — 可选兜底
